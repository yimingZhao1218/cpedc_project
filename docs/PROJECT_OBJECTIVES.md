# CPEDC 马口组 PINN 项目目标与实施路线图

## 📋 项目总体目标

### 核心问题
**川中震旦系马口组灰岩气藏**面临的工程难题：
1. **边水强侵**：边水沿裂缝-孔洞高渗带侵入，导致气井见水、产能下降
2. **强非均质性**：渗透率空间变化大（0.1-50 mD），传统均质假设失效
3. **井间干扰**：多井开发需要理解井间连通性，避免相互影响
4. **制度优化难**：传统数值模拟耗时长（2-4 小时），难以快速评估策略

### 解决方案
使用 **Physics-Informed Neural Networks (PINN)** 构建数字孪生系统：
- 替代传统网格数值模拟（效率提升 100 倍）
- 同时求解 PDE + 反演参数（k, p_wf, dp_wellbore）
- 提供实时预测预警与决策支持

### 最终交付物
1. **精准预测模型**：压力场 p(x,y,t) + 含水饱和度 Sw(x,y,t)，Test MAPE < 15%
2. **渗透率反演**：空间渗透率场 k(x,y) + 井间连通性矩阵 C_ij
3. **水侵预警系统**：提前 3-6 个月预测见水时间 + 风险排序
4. **制度优化工具**：3 种策略 PINN 推演，1 分钟完成全生命周期模拟
5. **Web 可视化平台**：Streamlit 交互式应用，支持评委展示和工程应用

### M5 两相井藏耦合同化训练（核心模块）
**目的**：训练 PINN 模型，同时拟合生产数据 + 满足物理 PDE

**输入**：
- 井位生产数据（qg, qw, p_wf）
- 物理参数（k, φ, μ, c, Bg）
- 配点样本（IC/BC/PDE 采样点）

**输出**：
- 训练好的 PINN 模型（可预测 p(x,y,t), Sw(x,y,t)）
- 反演参数：k_frac, dp_wellbore, (pwf_nets 权重)
- 训练历史：损失曲线、指标演化

**核心技术**：

#### 5.1 模型架构（`m5_model.py`）
- **Field Net**：7 层 MLP，输入 (x,y,t)，输出 (p, Sw)
- **k_net**：3 层子网络，输入 (x,y)，输出 k(x,y)
- **Fourier Features**：高频信号编码，提升拟合能力
- **Well Model**：Peaceman 井模型，耦合场变量与井位产量

**目的**：
- Field Net 学习 p(x,y,t) 和 Sw(x,y,t) 的时空演化
- k_net 反演空间渗透率场（捕捉非均质性）
- Peaceman 模型实现井藏耦合（qg = WI × (p_cell - p_wf) / Bg）

#### 5.2 损失函数（`assimilation_losses.py`）
**多目标优化**：
1. **L_qg**：产气量监督损失（数据驱动）
2. **L_PDE**：两相守恒方程残差（物理约束）
3. **L_IC**：初始条件（t=0 时 p=p0, Sw=Swc）
4. **L_BC**：边界条件（边界压力 p=p_boundary）
5. **L_reg**：正则化（k_net 平滑、Sw 边界约束）

**目的**：
- 平衡数据拟合与物理可靠性
- 避免过拟合（纯数据驱动会在测试集失效）
- 满足物理守恒定律

#### 5.3 训练策略（`m5_trainer.py`）
**核心技术**：

1. **ReLoBRaLo 自适应权重平衡**
   - 目的：自动调整 10 个损失项的权重比例
   - 方法：基于各损失项的相对学习速率动态平衡
   - 效果：避免某个损失项主导训练（如 L_qg 压制 L_PDE）

2. **RAR 残差驱动自适应采样**
   - 目的：在 PDE 残差大的区域增加采样密度
   - 方法：每 200 步评估残差，在前 20% 高残差区域额外采样
   - 效果：提高 PDE 求解精度，避免低残差区域浪费计算

3. **4 阶段 Curriculum Learning**
   - Stage A (0-15%)：IC 锁定（20×），PDE 极弱（0.01×），建立初始场
   - Stage B (15-40%)：PDE 0.1→1.0 线性增长
   - Stage C (40-75%)：PDE 1.0→5.0，强化物理约束
   - Stage D (75-100%)：维持 PDE=5.0，精细调优

   **目的**：
   - 先用 IC 建立合理初始场
   - 逐步引入 PDE 约束，避免训练早期崩溃
   - 后期强化物理一致性

4. **多参数组学习率**
   - field_net: base_lr
   - inversion (pwf_nets + dp_wellbore): base_lr × 1.5
   - k_frac: base_lr × 10（补偿梯度量级差异）
   - k_net: base_lr × 0.1 + warmup 2000 步（后期训练）

   **目的**：
   - 不同参数梯度量级差异大（k_frac 梯度小 100 倍）
   - 分组学习率确保所有参数都能有效更新

5. **学习率调度**
   - Warmup 1000 步（线性增长到 base_lr）
   - Cosine Decay（逐步衰减到 min_lr）
   - Stage D 额外 0.5× 衰减（减少后期震荡）

**对应代码**：
- `src/pinn/m5_trainer.py` — M5Trainer 类
- `src/pinn/relobralo.py` — ReLoBRaLo 权重平衡
- `src/pinn/rar_sampler.py` — RAR 自适应采样

### M6 连通性分析与渗透率反演（核心交付物）
**目的**：基于反演的 k(x,y) 场，分析井间流动通道和连通性

**输入**：
- 训练好的 PINN 模型（含 k_net）
- 井位坐标

**输出**：
1. **k(x,y) 场热力图**：80×80 网格分辨率
2. **连通性矩阵 C_ij**：井间归一化连通性系数 [0,1]
3. **主控流动通道**：Dijkstra 最短渗流路径（绿色曲线）
4. **Sw 空间演化**：4 个时间快照

**技术方法**：

#### 6.1 k(x,y) 场评估
```python
# 在规则网格上评估 k_net
k_map = model.k_net.get_k_mD(xy_grid)  # (80×80)
# 高斯平滑（σ=1.5）消除网格噪声
k_map = gaussian_filter(k_map, sigma=1.5)
```

**目的**：
- 可视化反演的空间渗透率分布
- 识别高渗透带（裂缝-孔洞发育区）

#### 6.2 连通性矩阵计算
```python
# 1. 构建网格图（节点=网格点，边权=渗流阻力）
resistance_ij = ds / (k_avg × h)

# 2. Dijkstra 最短路径算法
dist_matrix = dijkstra(graph, indices=well_nodes)

# 3. 连通性系数（阻力倒数）
C_ij = 1.0 / (dist_matrix + ε)

# 4. 归一化到 [0,1]
C_ij_norm = C_ij / sqrt(C_ii × C_jj)
```

**工程意义**：
- **C_ij 接近 1**：井 i 和井 j 高度连通，压力传播快，边水侵入优势路径
- **C_ij 接近 0**：两井间存在屏障（致密基质或断层遮挡）
- **主控通道**：最短渗流路径，指导排水采气井位优选

**应用价值**：
1. 识别边水侵入的优势路径
2. 指导井网调整（避开高连通区，减少相互干扰）
3. 优先在高连通井部署排水采气设备

**对应代码**：`src/pinn/connectivity.py` (ConnectivityAnalyzer 类)

**关键改进**：
- k 场高斯平滑（消除噪声）
- 通道路径 B-spline 平滑（从折线变平滑曲线）
- 工程解释自动生成（最强/最弱连通井对分析）

### M7 水侵预警与制度优化（核心交付物）
**目的**：用 PINN 做 forward simulation，预测水侵风险 + 优化生产策略

#### 7.1 水侵预警
**输入**：
- 训练好的 PINN 模型
- 井位坐标

**输出**：
1. **Sw(t) 时间序列**：各井含水饱和度演化
2. **风险指数 R_w(t)**：归一化水侵风险 = (Sw - Swc) / (1 - Swc - Sgr)
3. **见水时间预测**：Sw 超过阈值（0.40）的时刻
4. **风险排序表**：按优先级排序，指导措施部署

**风险等级定义**：
- **安全**（R_w < 0.15）：正常生产
- **预警**（0.15 ≤ R_w < 0.35）：加密监测
- **危险**（0.35 ≤ R_w < 0.60）：准备排水采气
- **水淹**（R_w ≥ 0.60）：立即排水或关井

**工程价值**：
- 提前 3-6 个月预测见水
- 为排水采气设备部署争取时间
- 指导生产制度调整

#### 7.2 制度优化（PINN-based forward simulation）
**目的**：快速评估不同生产策略下的累计产气 Gp 和水侵风险

**3 种策略**：

1. **稳产方案**（基线）
   - 策略：维持当前产量不变
   - 优点：累计产气最大（~100%）
   - 缺点：水侵最早，风险最高
   - 适用：短期开发评价（1-2 年）

2. **阶梯降产**（推荐 ⭐）
   - 策略：分 3 阶段降低产量（中期 p_wf +5 MPa，后期 p_wf +10 MPa）
   - 优点：累产 ~85% 稳产，见水延后 150 天
   - 平衡：产量与水侵的最优平衡
   - 适用：长期稳产（3-5 年）

3. **控压方案**（保守）
   - 策略：限制最低 p_wf = 45 MPa
   - 优点：水侵最慢，无水期最长
   - 缺点：累产仅 ~70% 稳产
   - 适用：边水强压区、保守开发

**技术创新**：

**传统方法**：简化乘数模型 `qg_new = qg_base × decay_factor`
- ❌ 缺乏物理基础
- ❌ 无法反映压差-产量关系
- ❌ Sw 演化与产量脱钩

**PINN 方法**（v3.1 改进）：基于 Peaceman 公式的真实压差-产量关系
```python
# 阶梯降产：提高 p_wf 降低压差
pwf_step = pwf_base.copy()
pwf_step[middle_period] += 5.0   # 中期 +5 MPa
pwf_step[late_period] += 10.0    # 后期 +10 MPa

# 压差比例 → 产量比例（Peaceman 公式 qg ∝ Δp）
dp_base = p_cell - pwf_base
dp_step = p_cell - pwf_step
qg_step = qg_base × (dp_step / dp_base)

# Sw 演化与产量耦合（产量越低，水侵越慢）
sw_step[i] = sw_step[i-1] + dsw_base × (dp_step / dp_base)
```

**验证逻辑**：
- 自动检查 Gp_稳产 > Gp_阶梯降产 > Gp_控压
- 如不满足，输出警告：检查 p_wf 扰动幅度

**工程价值**：
- **速度优势**：PINN 推演 1 分钟 vs 传统模拟 2-4 小时（效率提升 100 倍）
- **决策支持**：快速评估多种策略，支持敏感性分析
- **物理可靠**：基于真实压差-产量关系，非简化假设

**对应代码**：`src/pinn/water_invasion.py` (WaterInvasionAnalyzer 类)

**关键改进**（v3.1）：
- 策略评估从简化乘数改为 Peaceman 公式
- 3×3 矩阵对比图（行=策略，列=指标）
- 决策推荐矩阵（4 种开发目标 → 最优策略）

## 🔬 关键技术组件

### 1. Peaceman 井模型（核心创新）
**目的**：实现井藏耦合，建立 qg 与 p(x,y,t) 的物理联系

**数学模型**：
```
qg = WI × (p_cell - p_wf - dp_wellbore) / Bg

WI = 2π × k_frac × h / (μg × ln(r_e / r_w))
```

**参数反演**：
- **k_frac**：裂缝有效渗透率（v3.1 单参数，替代 k_eff × f_frac）
- **dp_wellbore**：井筒压降（管流摩阻）
- **pwf_nets**：每口井的 p_wf(t) 时间序列网络

**目的**：
- k_frac 作为全局参数，反映气藏整体渗透率
- dp_wellbore 补偿井筒摩阻损失
- pwf_nets 处理 p_wf 观测数据缺失或噪声

**对应代码**：`src/pinn/well_model.py` (PeacemanWellModel 类)

**关键修复**（v3.2）：
- k_frac 独立学习率（10× base_lr，补偿梯度小）
- 确保井源项 qg 对 k_frac 有梯度（不 detach）
- 防止 k_frac 梯度消失导致恒为初始值

---

### 2. ReLoBRaLo 自适应权重平衡
**目的**：自动调整 10 个损失项权重，避免手动调参

**原理**：
```python
# 基于各损失项的相对学习速率
λ_i(t) ∝ exp(τ × log(L_i(t) / L_i(t-1)))

# 避免某项主导训练
weights = softmax(λ_1, λ_2, ..., λ_10)
```

**效果**：
- 前期 L_qg 快速下降时，自动降低其权重
- 后期 L_PDE 收敛慢时，自动提高其权重
- 无需人工调整 10 个权重超参数

**对应代码**：`src/pinn/relobralo.py` (ReLoBRaLo 类)

---

### 3. RAR 残差驱动自适应采样
**目的**：在难拟合区域（PDE 残差大）增加采样密度

**算法**：
```python
# 每 refine_interval=200 步：
# 1. 在 4096 个点评估 PDE 残差
residual = |∂p/∂t - ∇²p - source|

# 2. 选择前 20% 高残差点
high_res_points = top_k(residual, k=0.2 × 4096)

# 3. 下次采样时，50% 来自高残差区，50% 均匀采样
x_pde = 0.5 × sample_from_rar_points() + 0.5 × uniform_sample()
```

**效果**：
- 井附近（残差高）采样密度 3× 背景区域
- PDE 精度提升 ~20%
- 训练步数减少 ~15%

**对应代码**：`src/pinn/rar_sampler.py` (RARSampler 类)

---

### 4. 两相守恒 PDE（物理约束）
**目的**：确保预测结果满足质量守恒定律

**气相守恒**：
```
φ × ∂(Sg × ρg)/∂t = ∇·(ρg × k × krg / μg × ∇p) + qg_well
```

**水相守恒**：
```
φ × ∂(Sw × ρw)/∂t = ∇·(ρw × k × krw / μw × ∇p) + qw_well
```

**实现简化**（假设液相不可压）：
```python
# 气相压力方程（主控方程）
residual_g = ∂p/∂t - α_x × ∂²p/∂x² - α_y × ∂²p/∂y² - source_g

# 水相饱和度方程（辅助方程）
residual_w = ∂Sw/∂t - β × (∂²Sw/∂x² + ∂²Sw/∂y²)

# 饱和度约束
Sg + Sw = 1
Swc ≤ Sw ≤ 1 - Sgr
```

**目的**：
- 物理约束防止过拟合（纯数据驱动在测试集失效）
- 确保场变量满足守恒定律（可外推到未来）
- 提供正则化效应（平滑场变量）

**对应代码**：`src/pinn/assimilation_losses.py` (compute_pde_residual 方法)

## 🧪 验证与评估

### 消融实验（Ablation Study）
**目的**：验证各技术组件的有效性

**6 组对比实验**：

1. **pure_ml**：纯数据驱动（关闭所有物理约束）
   - 目的：建立基线，量化物理约束的贡献
   - 预期：训练集拟合好，测试集 MAPE 高（过拟合）

2. **pinn_const_k**：单相 PDE + 常数 k
   - 目的：验证 PDE 约束的基本效果
   - 预期：比 pure_ml 泛化性好，但精度受限于均质假设

3. **pinn_knet**：单相 PDE + k_net 反演 k(x,y)
   - 目的：验证空间渗透率反演的必要性
   - 预期：捕捉非均质性，精度提升

4. **pinn_full**：完整模型（PDE + k_net + Fourier + RAR）
   - 目的：展示最优配置
   - 预期：Test MAPE 最低（< 15%）

## 💡 关键设计决策与权衡

### 决策 1：k_frac 单参数化（v3.1）
**原方案**：k_eff + f_frac（两参数）  
**新方案**：k_frac = k_eff × f_frac（单参数）

**理由**：
- 两参数梯度相互干扰，难以收敛
- 实际工程中只关心有效渗透率 k_frac
- 简化参数空间，提高训练稳定性

### 决策 2：分阶段 Curriculum Learning
**原方案**：固定权重训练到底  
**新方案**：4 阶段动态权重（IC 锁定 → PDE 逐步增强）

**理由**：
- 前期 IC 建立合理初始场，避免梯度爆炸
- 中期逐步引入 PDE，平稳过渡
- 后期强化物理约束，精细调优

### 决策 3：ReLoBRaLo vs 手动权重
**原方案**：手动调整 10 个损失项权重  
**新方案**：ReLoBRaLo 自适应平衡

**理由**：
- 手动调参需要数十次试错
- ReLoBRaLo 自动适应训练过程
- 减少人工干预，提高可复现性

### 决策 4：RAR vs 均匀采样
**原方案**：固定均匀采样  
**新方案**：RAR 残差驱动自适应采样

**理由**：
- 井附近 PDE 残差大，需要更多配点
- 远场残差小，减少采样节省计算
- 提升 PDE 精度 ~20%

### 决策 5：Streamlit vs 传统报告
**原方案**：静态 PDF 报告  
**新方案**：交互式 Web 应用

**理由**：
- 评委可现场调整参数（PVT 计算器）
- 交互式图表（Plotly 缩放/悬停）
- 技术亮点更突出
- 工程应用价值更直观
