# M1～M7 各阶段核心作用与实现方式

本文档说明 CPEDC 创新组「深层碳酸盐岩气藏 PINN 数字孪生系统」中 **M1～M7 每个阶段的核心作用** 与 **实现方式**。

---

## 总览

| 阶段 | 核心作用 | 关键实现入口 |
|------|----------|--------------|
| **M1** | 数据层与坐标统一 | `m1/m1_data_processor.py` |
| **M2** | 弱空间地质域构建 | `m2/m2_geo_builder.py` |
| **M3** | 物性模块（PVT + 相渗） | `physics/pvt.py`、`physics/relperm.py` |
| **M4** | PINN 增强基线（最小闭环） | `m4/run_m4_baseline.py`、`pinn/trainer.py` |
| **M5** | 井—藏耦合同化 + k(x,y) 反演 | `m5/run_m5_single_well.py`、`pinn/m5_trainer.py` |
| **M6** | 消融 + UQ + 连通性分析 | `m6/run_ablation_suite.py`、`pinn/connectivity.py` |
| **M7** | 水侵预警 + 制度优化 | `pinn/water_invasion.py` |

## M1：数据层与坐标统一

### 核心作用

- 为全流程提供**统一、可用的井与生产数据**：井眼 3D 轨迹、MK 组顶底界代表点、SY9 日生产数据。
- **坐标基准统一**：MSL 向上为正，$z = z_{\text{datum}} - \text{TVD}$，保证与 M2 地质域、PINN 配点一致。
- 产出数据质量报告，支撑后续 M2 边界/网格与 M4/M5 井数据采样。

### 实现方式

- **入口**：`main.py --stage m1` 调用 `M1_DataProcessor().run()`（`src/m1/m1_data_processor.py`）。
- **井眼轨迹**：最小曲率法由井斜数据计算 3D 轨迹，MD 单调性检查。
- **MK 段代表点**：基于分层数据 + 井斜插值得到顶底界位置，输出 `mk_interval_points.csv`。
- **SY9 生产数据**：缺失插补、时间连续性检查与清洗，输出 `production_SY9.csv`。
- **字段与单位**：统一字段名与单位（如压力 MPa、产量等），写入 `data/clean/`。
- **验收**：`validate_m1.py`（M1Validator）检查井位散点、MK 厚度、轨迹单调性、生产数据连续性。

### 主要输出

- `data/clean/wellpath_stations.csv`（井眼 3D 轨迹）
- `data/clean/mk_interval_points.csv`（MK 段代表点）
- `data/clean/production_SY9.csv`（SY9 日生产数据）
- M1 数据质量与验收报告、图件

## M2：弱空间地质域构建

### 核心作用

- 在平面上定义**模型边界**与**空间域**，为 PINN 提供有界求解域。
- 得到 **MK 顶底面与厚度场** $h(x,y)$，用于 2.5D 厚度加权 PDE 与井周加密。
- 生成 **PINN 配点网格**（域内 + 边界 + 井周加密），供 M4/M5/M6 采样。

### 实现方式

- **入口**：`main.py --stage m2` 调用 `M2_GeoDomainBuilder().run()`（`src/m2/m2_geo_builder.py`）。
- **边界**：基于井位凸包 + 缓冲区（Shapely）得到 `model_boundary.csv`。
- **顶底面**：用 M1 产出的 MK 代表点做 **Kriging 插值**（Pykrige），得到 `mk_top_surface.csv`、`mk_bot_surface.csv`。
- **厚度场**：$h(x,y) = z_{\text{top}}(x,y) - z_{\text{bot}}(x,y)$，输出 `mk_thickness.csv`。
- **配点网格**：在边界内生成均匀网格 + 井周加密，输出 `collocation_grid.csv`、`boundary_points.csv`。
- **验收**：`validate_m2.py` 检查边界包含井位、顶底厚等值图、交叉验证 MAE/RMSE、配点井周加密。

### 主要输出

- `geo/surfaces/mk_*_surface.csv`、`mk_thickness.csv`
- `geo/grids/collocation_grid.csv`、`boundary_points.csv`
- `geo/boundary/model_boundary.csv`
- M2 地质域与验收报告、图件

---

## M3：物性模块（PVT + 气水相渗）

### 核心作用

- 提供**天然气 PVT** 与 **气水相渗** 的**唯一权威 API**，供 M4/M5 的 PDE、井模型、反演使用。
- 建立 **M3→M4 数据管道**：PCHIP 拟合后导出多项式系数，供 TorchPVT 可微计算。
- 为 M5 提供参考点 (p_ref, T_ref) 下的 cg、Bg 及相渗端点（Swr、Sgr、krw_max、krg_max）。

### 实现方式

- **入口**：`main.py --stage m3` 中构建 `GasPVT(config)`、`RelPermGW(config)` 并做功能测试（`src/main.py` 的 `run_m3()`）。
- **PVT**（`physics/pvt.py`）：
  - 压力方向 **PCHIP**，温度方向线性插值，多温度（如 16.5～140.32 ℃）。
  - API：`z(p,T)`、`bg(p,T)`、`cg(p,T)`、`rho(p,T)`、`alpha_T(p,T)`、`query_all(p,T)`。
  - 导出系数：`export_all_polynomial_coeffs()` → 供 M4 TorchPVT 使用。
- **相渗**（`physics/relperm.py`）：
  - PCHIP 插值 + 端点/单调/非负约束，支持 `krw(Sw)`、`krg(Sw)` 及导数、`endpoints()`。
- **验收**：`m3/validate_m3.py`（M3Validator）调用 PVTValidator、RelPermValidator，出图并生成 M3_pvt_report、M3_relperm_report。

### 主要输出

- 可复用 API：GasPVT、RelPermGW（无文件，运行时使用）
- `outputs/.../figs/M3_pvt_curves.png`、`M3_relperm_curves.png`
- `outputs/.../reports/M3_pvt_report.md`、`M3_relperm_report.md`

## M4：PINN 增强基线（最小闭环）

### 核心作用

- 在**不反演 k(x,y)** 的前提下，训练一个满足**两相渗流 PDE** 且**拟合 SY9 井口压力**的 PINN，形成「物理 + 数据」的**最小闭环**。
- 为 M5 提供可加载的 checkpoint（`pinn_best.pt`、`pinn_final.pt`），供井藏耦合与 k 反演续训或初始化。
- 验证从 M1/M2/M3 到 PINN 的整条链路可跑通、可验收。

### 实现方式

- **入口**：`main.py --stage m4` 或 `python src/m4/run_m4_baseline.py`（推荐），依赖 M2 的 `collocation_grid.csv`。
- **网络**：输入 (x,y,t) → Fourier Feature Encoding（32 频率）→ ResNet [256×6] + LayerNorm + Tanh → 输出 p（tanh 有界）、Sw（Sigmoid ∈ (0,1)）；M4 基线**无 k_net**，渗透率为常数。
- **物性**：TorchPVT、TorchRelPerm（可微），物性系数可来自 M3 导出。
- **损失**：$\mathcal{L} = \lambda_{\text{IC}}\mathcal{L}_{\text{IC}} + \lambda_{\text{BC}}\mathcal{L}_{\text{BC}} + \lambda_{\text{PDE}}\mathcal{L}_{\text{PDE}} + \lambda_{\text{data}}\mathcal{L}_{\text{data}} + \lambda_{\text{sw\_phys}}\mathcal{L}_{\text{sw\_phys}}$（IC/BC/PDE/井口压力/Sw 物理弱约束）。
- **训练**：`pinn/trainer.py`（PINNTrainer）分阶段 A→B→C→D（由 `m4_config.training_stages` 驱动），Warmup + Cosine 学习率。
- **验收**：`m4/validate_m4.py`，SY9 压力 MAPE、Sw 范围等，输出 M4_validation_report。

### 主要输出

- `outputs/.../ckpt/pinn_best.pt`、`pinn_final.pt`
- `outputs/.../figs/M4_training_history.png`、`M4_pressure_comparison.png`
- `outputs/.../reports/M4_validation_report.md`

## M5：井—藏耦合同化 + k(x,y) 反演

### 核心作用

- 在 M4 基线上引入**井模型**（Peaceman、WHP→p_wf 可学习井筒压差）、**完整两相守恒 PDE** 与**井源项**，实现井—藏耦合。
- **反演**：$k_{\text{eff}}$、$f_{\text{frac}}$、$\Delta p_{wb}$、**k(x,y) 场**（PermeabilityNet），并做 TV/Laplacian/井点先验正则化。
- 修复 Bg 等物性在井模型中的使用，统一采用 TorchRelPerm，为 M6 连通性、M7 水侵提供反演后的 k 场与 PINN 状态。

### 实现方式

- **入口**：`main.py --stage m5` 或推荐 `python src/m5/run_m5_single_well.py --well SY9 --steps 50000 --device cuda`。
- **模型**：`pinn/m5_model.py`（M5PINNNet = 场网络 + PermeabilityNet + 井模型），`pinn/well_model.py`（Peaceman、高斯源项、p_wf 隐变量）。
- **损失**：`pinn/assimilation_losses.py` — 完整两相 PDE、$\mathcal{L}_{q_g}$、$\mathcal{L}_{\text{WHP}}$、先验正则；ReLoBRaLo 自适应损失权重。
- **训练**：`pinn/m5_trainer.py`，可选 AMP、torch.compile（Linux）；M3 PVT 用于参考点 cg/Bg 覆盖先验。
- **后处理**：训练结束后可自动调用 M6 连通性分析与 M7 水侵预警（`generate_report()`）。

### 主要输出

- `outputs/.../ckpt/m5_pinn_*.pt`
- `outputs/.../figs/M5_qg_comparison_*.png`、`M5_pwf_inversion_*.png`、`M5_training_history.png`、`M5_pde_residual_map.png`
- `outputs/.../reports/M5_validation_report.md`、`M5_inversion_params.json`、`resolved_config_m5.json`

## M6：消融 + UQ + 连通性分析

### 核心作用

- **消融实验**：对比 5 组配置（pure_ml、pinn_const_k、pinn_knet、pinn_full、pinn_no_fourier），评估各组件贡献。
- **不确定性量化（UQ）**：多种子 ensemble + 多维度扰动（k 先验、p_boundary、f_frac），输出 P10/P50/P90 与参数后验。
- **连通性分析**：基于反演 k(x,y) 场构建井间渗流阻力图，Dijkstra 最短路径得到 $C_{ij}$ 与主控通道，支撑开发与水侵解释。

### 实现方式

- **入口**：`main.py --stage m6`；或 `python src/m6/run_ablation_suite.py`、`python src/m6/run_uq_ensemble.py`；M5 完成后 `generate_report()` 会自动调用连通性分析。
- **消融**：override 机制切换 pure_ml / pinn_const_k / pinn_knet / pinn_full / pinn_no_fourier，每组建议 ≥20000 步。
- **UQ**：`pinn/uq_runner.py`，多种子 + 先验扰动，汇总 P10/P50/P90。
- **连通性**：`pinn/connectivity.py` — 网格图边权 $1/(k\cdot h)$，Dijkstra → $C_{ij}$、主控通道；可选 XPINN/APINN（`m6_config.domain_decomposition`）。

### 主要输出

- `outputs/.../figs/M6_ablation_*.png`、`M6_uq_*.png`、`M6_k_field_channels.png`、`M6_connectivity_matrix.png`、`M6_sw_evolution.png`
- `outputs/.../reports/M6_ablation_report.md`、`M6_uq_report.md`、`M6_connectivity_matrix.csv`

## M7：水侵预警 + 制度优化

### 核心作用

- **水侵风险**：用 PINN 推理的 $S_w$ 时间序列计算水侵风险指数 $R_w(t) = (S_w(x_w,t) - S_{wc})/(1 - S_{wc} - S_{gr})$。
- **见水时间**：各井 $S_w$ 超过设定阈值（如 0.4）的时刻预测。
- **制度优化**：以 PINN 为前向模拟器，对比**稳产**、**阶梯降产**、**控压**等方案下的 $G_p$、$S_w$、$p_{wf}$，给出建议。

### 实现方式

- **入口**：M5 训练完成后 `generate_report()` 自动调用；或单独 `WaterInvasionAnalyzer(model, sampler, config).generate_all(output_dir)`（`pinn/water_invasion.py`）。
- **风险与见水**：在井点采样 (x_w, t)，用训练好的 PINN 得到 $S_w(x_w,t)$，算 $R_w(t)$ 与见水时刻。
- **制度优化**：在同一 PINN 上施加不同产量/压力边界，积分得到各策略下的累计产气、$S_w$ 演化、$p_{wf}$，出对比图与文字建议。

### 主要输出

- `outputs/.../figs/M7_water_invasion_dashboard.png`（$S_w(t)$、$R_w(t)$、见水时间、压力-Sw 双轴）
- `outputs/.../figs/M7_strategy_comparison.png`（稳产/阶梯/控压对比）
- `outputs/.../reports/M7_water_invasion_report.md`

## 依赖关系小结

```
M1（数据） → M2（地质域）
                 ↓
            M3（物性）—→ M4（PINN 基线）
                 ↓              ↓
                 └────────────→ M5（井藏耦合 + k 反演）
                                    ↓
                               M6（消融/UQ/连通性）
                               M7（水侵预警/制度优化）
```

- **M4** 依赖 M2（配点网格）、M3（物性系数可选）。
- **M5** 依赖 M2、M4 checkpoint（或从 M5 续跑），使用 M3 的 cg/Bg/相渗端点。
- **M6** 依赖 M2 配点；连通性分析依赖 M5 反演得到的 k(x,y)。
- **M7** 依赖 M5 训练好的 PINN 与采样器，无需单独跑 M6。
